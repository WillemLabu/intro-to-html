What is HTML?

HyperText Markup Language

It's the language of web pages.

It consists of a bunch of tags.

It describes content.

It is not the content.

HTML is not the content.

Content is what goes between the HTML tags.

How web servers work

Basic Rules

How to do stuff with HTML

Describe the content


  <p>
    This is a paragraph of text, wrapped in "p" tags, which means
    this is.. a paragraph, or a sentence of text that is human-
    (and, importantly, machine-) readable!
  </p>

Blocks go in blocks


  <header>
    <h1>This is a heading</h1>
    <p>All <hx /> tags are blocks elements!</p>
  </header>

Inline tags go in blocks


  <article>
    <h1>Hacker News</h1>
    <p>All <a href="https://news.ycombinator.com/">the freshest news</a>
       for your face!</p>
  </article>

Example: Click here!

Block tags do not go inside inline elements


  <a href="https://news.ycombinator.com/">
    <h1>Hacker News</h1>
    <p>All the freshest news for your face!</p>
  </a>

Please, don't do this.

Close nested elements


  <p>This is a properly <b>bolded</b> word!</p>

  <p>This one, not so <b>much</p>.</b>

Semantics

(from Ancient Greek: σημαντικός sēmantikós, "significant") is the study of meaning.

It focuses on the relation between signifiers, like words, phrases, signs, and symbols, and what they stand for; their denotation.

Not just humans

Websites are not just read by humans. Much more often, in fact, will they be "read" by machines.

Your browser needs to parse the HTML before it will be rendered and displayed.

Search Engine Crawlers will read sites a few times a week (or even multiple times a day) to index content.

Semantic code is clean

More often than not, you will be rewriting, or even copying code.

Clean code is easy to maintain and update.

What is Git?

It's a Version Control System.

Never lose work again! Save every file, and every step along the way, of your entire project.

It allows you to work with other people on the same code, without overwriting each other's work.

How does Git work?

There are three (conceptual) areas in any Git project:

  • Repository
  • Working Directory
  • Staging Area
